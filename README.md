# FluxVita All-in-One

公司统一工作空间。把飞书、GitLab、Linear、邮件等所有工具的数据沉淀到一个地方，让 AI Agent 拥有完整的公司上下文，帮你真正把事情做完。

---

## 它解决什么问题

日常工作里信息散落在各处——飞书里有讨论，GitLab 里有代码，Linear 里有任务，邮件里有决策。每次让 AI 帮忙，都要手动复制粘贴背景信息，效果打折。

All-in-One 把这些数据统一接入，Agent 在回答你的问题、帮你执行任务时，能直接调取公司真实的数据，不靠猜测。

---

## 核心功能

### AI Agent 对话
打开对话界面，直接描述你想做的事。Agent 会根据需要自动查询数据、搜索记录、调用工具，给出有依据的回答或直接帮你完成任务。

**可以问的问题举例：**
- "上周的 GitLab MR 里有没有涉及支付模块的改动？"
- "帮我整理 Linear 里本周新增的 Bug，按优先级列出来"
- "飞书里关于用户增长的讨论，最近有什么结论？"
- "我的邮件里有没有未回复的重要邮件？"

### 数据连接器
自动同步公司各平台数据，始终保持最新：

| 平台 | 同步内容 |
|------|---------|
| 飞书 | 群消息、文档、组织架构、日历 |
| GitLab | 代码仓库、MR、Issue、提交记录 |
| Linear | 任务、项目、团队进度 |
| PostHog | 用户行为数据、事件统计 |
| Figma | 设计文件、组件库 |
| 邮件 | 收发邮件记录 |

### 记忆库
Agent 会记住重要结论、你的工作偏好、常用背景信息，下次对话直接沿用，不用重复解释。

### 管理后台
查看数据同步状态、管理团队成员权限、配置连接器。

## 开始使用

### 下载客户端

**macOS：**
前往 [Releases 页面](https://gitlab.fluxvitae.com/Aiden/allinone/-/releases) 下载最新版 `.dmg`，拖入应用程序文件夹即可。

**Windows：**
查看 `scripts/Windows打包说明.md`。

### 首次配置

1. 打开应用，填入 Gateway 地址（联系 Aiden 获取）
2. 使用公司账号登录
3. 开始对话

## 使用场景

**每日晨会准备**
"帮我汇总昨天的进展：Linear 里完成了什么任务，GitLab 有哪些 MR 合并了，飞书有没有重要讨论。"

**排查问题**
"线上昨天下午 3 点左右有没有异常？结合 PostHog 数据和 GitLab 当天的部署记录分析一下。"

**整理信息**
"把关于新功能设计的飞书讨论和 Linear 任务关联起来，梳理成一个简洁的需求说明。"

**跨工具查询**
"这个功能最早是谁提出来的？当时的讨论在哪里？"
